## 書籍紛失検知システム

### 使用技術
- Python
- MeCab
- GCP(VisionAPI,BooksAPI)

### ファイルの場所
./file/book

### 背景
書店には書籍の盗難防止システムがあることが多いですが、
職場や研究室など共同で書籍を使う環境においてはそれがなく、
書籍が無くなったことに気づきづらいため、
書籍が無くなったことを検知するプログラムを開発しました。
大学の卒業研究です。

### 詳細
全ての本が揃っている状態(A)の本棚の写真、
全ての本のISBNリスト、
調査対象(B)の本棚の写真
この3つの画像を引数として与えます。

すると、写真に映っている文字を認識し、
それを単語ごとに分解してデータベースを作成します。

次に、AとBの差分を求め、紛失した書籍に含まれる単語を求めます。

最後に、ISBNリストから取得した書籍タイトル・著者・出版社の情報と、
紛失書籍の単語リストを照らし合わせることで、書籍ごとに「紛失度」を求めます。

単語の出現頻度が少ない単語に大きな重みをつけることで、
「数学1」「数学2」「数学3」「数学A」「数学B」「物理A」
という本棚から「数学3」が無くなった場合でも、
"3"に大きな重みをつけているため、「数学1」よりも「数学3」の方が紛失度が高くなります。

## 今日の日付を「yyyymmdd(曜)」形式に変換してフォルダ内のファイルを開くバッチファイル

### 使用技術
- Windowsスクリプト

### ファイルの場所
なし(現職場で作成したがデータの持ち出しが禁止されているため)

### 背景
業務で1日何度も開くチェックシートがあるのですが、

G:\abc\abc\abc\yyyymmdd(曜)\abc\abc\チエックシート.xlsx に入っています。

例えば2024年8月1日なら、

G:\abc\abc\abc\¥20240801(木)\abc\abc\チェックシート.xlsx といった具合です。ここで

G:\abc\abc\abc\

をクイックアクセスに入れたとしても、
そのあと3フォルダ下のファイルを開かなくてはいけないので、
少しではありますが手間がかかります。
その手間を解消したくて作りました。

### 詳細
localdatetimeで求めた時間を適切に切り出し、
dayofweekで求めた曜日(数字)を変換して、
適切なパスとなるよう組み合わせてstartします。
夜勤のときは翌日のチェックシートも使うため、
翌日のチェックシートを開く.batも作成しました。



## Windowsからエアコンや電気を操作するCLIプログラム

### 使用技術
- Python
- SwitchBotAPI

## ファイルの場所
https://github.com/sen5q/switchbot-for-pc

### 背景
SwitchBotハブミニと呼ばれるIoT製品があります。
これは、赤外線信号を記憶・模倣することができる製品であり、
エアコンやテレビなどの赤外線通信を行う家電をスマートフォンから制御できる製品です。

家電それぞれの純正リモコンを使う必要がなくなるためとても便利なのですが、
公式からはスマートフォンアプリしかリリースしておらず、
Windowsやmacから操作できたらいいなあと思い、制作を開始しました。

### 詳細
公式アプリから取得できる認証情報をtoken.pyに入れ、
get_device_id.pyとget_scene.pyを実行し、
取得したデバイスIDとシーンIDをtoken.pyに入力します。そのあと、

'$ airconditioner.py

と実行するとエアコンを操作することができます。

第一引数には冷房、暖房、オフを意味するc,h,oのいずれかを、
第二引数には温度を与えることで、引数通りにエアコンを操作できます。

同様に、電気を操作する light.py と、
事前にアプリの方で作成してあるシーンを実行する scene.py も作成しました。

このプログラムを作ろうと思った数か月前にAPIがパージョンアップし認証方法が変更したことで、
使える情報が公式のドキュメントあるいは数えるほどしかない個人ブログだったと記憶しています。
自分が行いたい操作は個人ブログを読む限り実現できなかったので、公式のドキュメントを読み解きました。

## ボイスチャンネルにいる複数のユーザーをまとめて移動させるDiscordBot

### 使用技術
- Python
- Discord
- Linux

### ファイルの場所
•/file/discord

### 背景
Discordというコミュニケーションツールには、いくつかのボイスチャンネルを設定できます。
ZOOMのブレイクアウトルームみたいなものです。

ここで、ボイスチャンネルAにいる特定の人をボイスチャンネルBに移動させる機能はあるのですが、
ボイスチャンネルAにいるすべての人をボイスチャンネルBに移動させることはできる機能はありません。
それが不便だったので作りました。

### 詳細
スラッシュコマンドに対応したDiscordBotとして作成しました。

/moveAB

と実行すると、Aにいる全ての人をBに移動できます。今後は

/gatherA

と入力することで、A以外にいる全ての人をAに移動させるコマンドを実装しようと思っています。

また、Linuxのレンタルサーバーを借り、そこにファイルをアップロードしているため、Botは常時起動しています。
